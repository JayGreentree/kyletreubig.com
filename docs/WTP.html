1 Introduction

1.1 Purpose of This Document

The purpose of this document is to describe a set of test steps that will be used to verify that kyletreubig.com has been designed to meet all of its requirements.

Top

1.2 Scope of This Document

This document contains procedures for running unit tests, integration tests, and customer acceptance tests. Procedures are described by steps and expected results.

Top

1.3 Description of This Document

The following section contains the test procedures for unit tests, integration tests, and customer acceptance tests. The requirements matrix and requirements specification are provided in the Appendix.

2.2.3 User Management

  1. Log in as user
    1. Proceed to 'login' page

      Should see user login form

    2. Fill in form with valid login credentials
    3. Submit the form

      Should see confirmation message

    4. Proceed to 'login' page

      Should see 'already logged in' message

  2. Log out
    1. Log in as a user
    2. Proceed to 'logout' page

      Should see confirmation message

    3. Proceed to 'login' page

      Should see user login form

  3. Modify own user information
    1. Log in as a user
    2. Proceed to 'edit my info' page

      Should see a user information form with own information

    3. Change the username to some other value
    4. Submit the form

      Should see access denial message

    5. Proceed to 'edit my info' page

      Should see a user information form with own information

    6. Change some value (other than password)
    7. Submit the form

      Should see confirmation message

    8. Proceed to 'view my info' page

      Should see updated user information

  4. Change own password
    1. Log in as a user
    2. Proceed to 'edit my info' page

      Should see a user information form with own information

    3. Change the password to new value
    4. Submit the form

      Should see confirmation message

    5. Logout
    6. Login with new password

      Should see confirmation message and be logged in

  5. Create a new user
    1. Log in as an admin user
    2. Proceed to 'add user' page

      Should see a user information form

    3. Fill in form with values:
      • Username: testuser
      • Password: testpass
    4. Submit the form

      Should see confirmation message

    5. Proceed to 'user list' page

      Should see testuser in user list

  6. Modify other user information
    1. Attempt to modify other user

      Should see access denial message

    2. Log in as an admin user
    3. Proceed to 'user list' page

      Should see testuser in user list

    4. Select testuser to edit

      Should see a user information form with testuser's information

    5. Change the values:
      • Username: moduser
    6. Submit the form

      Should see confirmation message

    7. Proceed to 'user list' page

      Should see moduser in user list

  7. Change user role
    1. Log in as an admin user
    2. Proceed to 'user list' page

      Should see own user in user list

    3. Select own user to edit

      Should see user info form with own user

    4. Change own user's role to 'user'
    5. Submit the form

      Should see access denial message

    6. Proceed to 'user list' page

      Should see moduser in user list

    7. Select moduser to edit

      Should see user info form with moduser

    8. Change moduser's role to 'admin'
    9. Submit the form

      Should see confirmation message

    10. Proceed to 'user list' page

      Should see moduser in user list as an admin user

  8. Delete user
    1. Log in as an admin user
    2. Proceed to 'user list' page

      Should see own user in user list

    3. Select own user to delete

      Should see a 'confirm deletion' popup or message

    4. Confirm the deletion

      Should see access denial message

    5. Proceed to 'user list' page

      Should see moduser in user list

    6. Select moduser to delete

      Should see a 'confirm deletion' popup or message

    7. Confirm the deletion

      Should see confirmation message

    8. Proceed to 'user list' page

      Should not see moduser in user list

2.2.4 Content Management

  1. Modify page contents
    1. Log in as an admin user
    2. Proceed to 'page list' page

      Should see list of editable pages

    3. Select a page to edit

      Should see page editor form with page contents

    4. Insert an image

      Should see a list of images contained in the upload folder

    5. Select an image

      Should see image tag inserted into page content with path to selected image file

    6. Make changes to the page contents
    7. Submit changes

      Should see confirmation message

    8. Proceed to front-end location of modified page

      Should see changes to the page contents

  2. View page versions
    1. Log in as an admin user
    2. Proceed to 'page list' page

      Should see list of editable pages

    3. Select a page to edit and modify its contents

      Should be same page as used in test 2.2.4.1

      Should see confirmation message

    4. Proceed to 'page list' page

      Should see modified page in page list

    5. Select modified page and view it's history

      Should see list of revisions

  3. Compare page versions
    1. Log in as an admin user
    2. Proceed to 'page list' page

      Should see list of editable pages

    3. Select a page and view it's history

      Should be same page as used in test 2.2.4.2

      Should see list of revisions

    4. Select the initial and head versions
    5. Click on 'compare'

      Should see page contents with deletions and additions highlighted

  4. Inspect about page
    1. Log in as an admin user
    2. Proceed to 'page list' page

      Should see about page in page list

  5. Uploaded content
    1. Log in as an admin user
    2. Proceed to asset management page

      Should see list of uploaded files, if any

    3. Create a new folder

      Should see a form for folder name

    4. Submit the form

      Should see confirmation message

    5. View contents of folder

      Should be folder just created

      Should see an empty folder

    6. Upload a file

      File should be one of the allowed file types set in the config file

      New filename should be left blank

    7. Submit the form

      Should see confirmation message

      Should see uploaded file inside folder

    8. Upload a file

      New filename should be changed

    9. Submit the form

      Should see confirmation message

      Should see uploaded file with new name inside folder

    10. Delete file

      Should be one of the files uploaded into folder

      Should see confirmation form

    11. Select no

      Should see folder with both files

    12. Delete file

      Should be one of the files uploaded into folder

      Should see confirmation form

    13. Select yes

      Should see confirmation message

      Should see folder with one file

    14. Delete folder

      Should be the folder created

      Should see confirmation form

    15. Select yes

      Should see confirmation message

      Should see parent folder without the test folder

2.2.5 Blog

All tests are executed against a test category and article created in the first few integration tests.

  1. Create new category
    1. Log in as an admin user
    2. Proceed to 'create category' page

      Should see a category name form

    3. Fill in form with category name, "tests"
    4. Submit the form

      Should see a confirmation message

    5. Proceed to 'category list' page

      Should see "tests" category in category list

  2. Rename category
    1. Log in as an admin user
    2. Proceed to 'category list' page

      Should see the test category in category list

    3. Select the test category to edit

      Should see a category name form with "tests"

    4. Change the name to "Integration Tests"
    5. Submit the form

      Should see confirmation message

    6. Proceed to 'category list' page

      Should see "Integration Tests" category in category list

  3. Create new article
    1. Log in as an admin user
    2. Proceed to 'create article' page

      Should see article editor form

      Should see a file upload form

    3. Fill in form with values:
      • Title: An Integration Test Article
      • Category: Integration Tests
      • Tags: testing, processes
      • State: draft
    4. Create a test article in a local file
    5. Select the local file in the file upload form
    6. Submit the form

      Should see confirmation message

    7. Proceed to 'article list' page

      Should see the test article in article list

    8. Proceed to back-end preview of new article

      Should see test article in rendered state

  4. Edit draft article
    1. Log in as an admin user
    2. Proceed to 'article list' page

      Should see list of articles

    3. Select the test article to edit

      Should see article editor form with test article

    4. Make changes to the article contents
    5. Submit changes

      Should see confirmation message

    6. Proceed to back-end preview of the test article

      Should see changes to article contents

  5. Publish article
    1. Log in as an admin user
    2. Proceed to 'article list' page

      Should see list of articles

    3. Select the test article to edit

      Should see article editor form with test article

    4. Change the state to published
    5. Submit changes

      Should see confirmation message

    6. Proceed to front-end location of article

      Should see test article in rendered state

  6. Edit published article
    1. Log in as an admin user
    2. Proceed to 'article list' page

      Should see list of articles

    3. Select the test article to edit

      Should see article editor form with test article

    4. Make changes to the article contents
    5. Submit changes

      Should see confirmation message

    6. Proceed to front-end location of article

      Should see changes to article contents

  7. Post a comment
    1. Log in as a user
    2. Proceed to front-end location of test article

      Should see test article with comment form. Comment form should only have subject and body fields.

    3. Log out
    4. Proceed to front-end location of test article

      Should see test article with comment form. Comment form should have name, email, etc. fields.

    5. Fill in form with values:
      • Name: Tester
      • Email: test@kyletreubig.com
      • Subject: A Test Comment
    6. Submit the form

      Should see confirmation message

    7. Proceed back to front-end location of article

      Should see newly posted comment in comment list of article

  8. Modify comment
    1. Log in as an admin user
    2. Proceed to front-end location of test article

      Should see test comment in comment list

    3. Select test comment to edit (click on 'edit' link)

      Should see comment form with test comment

    4. Modify the values:
      • Name: Test User
      • Subject: A Modified Comment
    5. Submit the form

      Should see confirmation message

    6. Proceed back to front-end location of article

      Should see test comment with changes

  9. Delete comment
    1. Log in as an admin user
    2. Proceed to front-end location of test article

      Should see test comment in comment list

    3. Select comment to delete (click on 'delete' link)

      Should see a 'confirm deletion' popup or message

    4. Confirm the deletion

      Should see confirmation message

    5. Proceed back to front-end location of article

      Should not see the test comment in comment list

  10. View articles by category
    1. Proceed to front-end category list

      Should see list of categories

    2. Select "Integration Tests" category

      Should see list of articles belonging to the category

      Should see the test article in the list

  11. View articles by tag
    1. Proceed to front-end tag list

      Should see list of tags

    2. Select "testing" tag

      Should see list of articles tagged with "testing"

      Should see the test article in the list

    3. Select "processes" tag

      Should see list of articles tagged with "processes"

      Should see the test article in the list

  12. Delete category
    1. Log in as an admin user
    2. Proceed to 'category list' page

      Should see the test category in category list

    3. Select the test category to delete

      Should see a 'confirm deletion' popup or message

    4. Confirm the deletion

      Should see confirmation message

    5. Proceed to front-end category list

      Should not see test category in category list

    6. Select "uncategorized" category

      Should see the test article in the list

  13. Check article view count
    1. Log in as an admin user
    2. Proceed to 'article statistics' page

      Should see list of articles. Note the view count of the test article.

    3. Proceed to front-end location of test article
    4. Proced to 'article statistics' page

      Should see list of articles. The view count of the test article should be incremented.

  14. Check popular articles
    1. Log in as an admin user
    2. Create a new article and publish it
    3. Proceed to 'article statistics' page

      Should see list of popular articles ordered by most popular. The test article should be more popular than the new article.
      Note the view counts of the test article.

    4. Proceed to the front-end location of the new article

      Repeat for at least as many times as the view counts of the test article

    5. Proceed to the 'article statistics' page

      Should see list of popular articles ordered by most popular. The new article should be more popular than the test article.

  15. Archive article
    1. Log in as an admin user
    2. Proceed to 'article list' page

      Should see list of articles

    3. Select the test article to edit

      Should see article editor form with test article

    4. Change the state to archived
    5. Submit changes

      Should see confirmation message

    6. Proceed to 'archived articles' page

      Should see test article in the article list

    7. Proceed to front-end article list

      Should not see test article in article list

  16. View RSS feed
    1. Ensure there are draft and published articles in the database
    2. Proceed to front-end RSS feed

      Should see up to 10 published articles in xml format

    3. Use an RSS feed reader to view RSS feed

      Should see up to 10 published articles in the reader

  17. Comment email report
    1. Ensure there are comments in the database with timestamps within the past 24 hours, and timestamps over 24 hours old
    2. Set up cron job to run the comment report task

      Should receive email with comments

      Should only be comments with timestamps within the past 24 hours

  18. Disable comments
    1. Log in as an admin user
    2. Proceed to 'article list' page

      Should see list of articles

    3. Select an article to edit

      Should be an article with active comments

      Should see article editor form with selected article

    4. Check the disable comments checkbox
    5. Submit changes

      Should see confirmation message

    6. Proceed to front-end location of article

      Should see comments on article, but no comment form

    7. Modify an article's database record

      Should be an article with active comments

      Should set the timestamp to some date over 90 days ago

    8. Proceed to front-end location of article

      Should see comments on article, but no comment form

2.2.6 Music Portfolio

  1. Create portfolio entry
    1. Log in as an admin user
    2. Proceed to 'create portfolio entry' page

      Should see entry editor form

      Should see a file upload form

    3. Fill in form
    4. Select mp3 file in file upload form
    5. Submit the form

      Should see confirmation message

    6. Proceed to front-end location of entry

      Should see submitted portfolio entry with audio clip

    7. Play the audio clip

      Should hear the audio file uploaded at entry creation

  2. Upload multiple files
    1. Log in as an admin user
    2. Proceed to 'portfolio entry list' page

      Should see list of entries

    3. Select an entry to edit

      Should be an entry with an audio clip

      Should see entry form with file upload form

    4. Select an mp3 file in file upload form
    5. Submit the form

      Should see confirmation message

    6. Repeat with a third mp3 file
    7. Proceed to front-end location of entry

      Should see submitted portfolio entry with three audio clips

  3. Remove uploaded file
    1. Log in as an admin user
    2. Proceed to 'portfolio entry list' page

      Should see list of entries

    3. Select an entry to edit

      Should be an entry with audio clips

      Should see entry form

      Should see list of associated audio clips

    4. Select an audio clip to delete

      Should see confirmation form

    5. Select no

      Should see the entry modification form

    6. Select an audio clip to delete

      Should see confirmation form

    7. Select yes

      Should see confirmation message

    8. Proceed to front-end location of entry

      Should see submitted portfolio entry with one less audio clip

  4. Delete a portfolio entry
    1. Log in as an admin user
    2. Proceed to 'portfolio entry list' page

      Should see list of entries

    3. Select an entry to delete

      Should see confirmation form

    4. Select no

      Should see list of entries

    5. Select an entry to delete

      Should see confirmation form

    6. Select yes

      Should see confirmation message

      Associated audio clips should be removed from the database

  5. Post a comment
    1. Log in as a user
    2. Proceed to front-end location of portfolio entry

    3. Log out
    4. Proceed to front-end location of portfolio entry

      Should see entry with comment form. Comment form should only have name, email, etc. fields.

    5. Fill in form with values:
      • Name: Tester
      • Email: test@kyletreubig.com
      • Subject: A Test Comment
    6. Submit the form

      Should see confirmation message

    7. Proceed back to front-end location of portfolio entry

      Should see newly posted comment in comment list of entry

2.2.7 Comments

  1. Classify comments
    1. Ensure logging level includes debug
    2. Manually insert into the database:
      • A comment marked as approved
      • Four comments flagged to be moderated
      • A comment marked as spam
    3. Log in as an admin
    4. Proceed to comment moderation queue page

      Should see four comments

    5. Approve a comment, selecting the "learn as ham" option

      Should see confirmation message

      Comment should no longer be in list

      6. Log should have entry about learning the comment as ham

    6. Approve a comment, selecting the "unlearn as spam" option

      Should see confirmation message

      Comment should no longer be in list

      7. Log should have entry about unlearning the comment as spam

    7. Mark a comment as spam, selecting the "learn as spam" option

      Should see confirmation message

      Comment should no longer be in list

      8. Log should have entry about learning the comment as spam

    8. Mark the remaining comment as spam, selecting the "unlearn as ham" option

      Should see confirmation message

      Comment should no longer be in list

      9. Log should have entry about unlearning the comment as ham

    9. Proceed to posted comments list

      Should see three comments

    10. Mark a comment as spam (no option selected)

      Should see confirmation message

      Comment should no longer be in list

      11. Log should have no entries regarding the Bayesian filter's learning

    11. Proceed to spam comments list

      Should see four comments

    12. Approve a comment (no option selected)

      Should see confirmation message

      Comment should no longer be in list

      13. Log should have no entries regarding the Bayesian filter's learning
